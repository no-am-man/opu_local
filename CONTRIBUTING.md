# Contributing to OPU

Thank you for your interest in contributing to the Orthogonal Processing Unit (OPU) project!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/no-am-man/opu_local.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit your changes: `git commit -m 'Add some feature'`
7. Push to the branch: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

1. Install dependencies:
```bash
pip install -r requirements.txt
```

2. Run the OPU to test:
```bash
python main.py
```

## Code Style

- Follow PEP 8 style guidelines
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and modular

## Testing

Before submitting a PR, please:
- Test your changes locally
- Ensure the OPU runs without errors
- Verify state persistence works (if you modified related code)
- Check that audio processing still functions correctly

## Pull Request Process

1. Update the CHANGELOG.md with your changes
2. Ensure your code follows the project's style
3. Make sure all tests pass (if applicable)
4. Update documentation if needed
5. Submit your PR with a clear description

## Areas for Contribution

- Performance optimizations
- Additional visualization features
- Enhanced phoneme recognition
- Documentation improvements
- Bug fixes
- New features that align with the OPU architecture

## Questions?

Feel free to open an issue for questions or discussions about contributions.
